# GDR Quality Standard

## Hard Requirements
- [ ] Source coverage reaches the intended daily input set
- [ ] Zero hallucinated sources
- [ ] Zero template leakage from source notes
- [ ] Zero repeated filler phrases across sections
- [ ] The opening thesis sentence is grounded in the day’s real context
- [ ] All inserted wikilinks are validated before inclusion

## Quantitative Targets
| Metric | Minimum | Good | Excellent |
|--------|---------|------|-----------|
| Sources found | 4+ | 6+ | 8+ |
| Validated wikilinks | 8 | 15 | 24 |
| Cross-source connections | 2 | 4 | 5+ |
| Output lines | 60+ | 100+ | 125+ |

## Cross-Source Synthesis Pattern
The key-themes paragraph should:
1. Open with a single thesis sentence naming the day’s central tension
2. Reference the day’s context in a concrete but generic way
3. Connect at least two sources in synthesis sentences rather than isolated bullets
4. End with a strong unifying theme label

## Per-Source Section Pattern
Each section should have:
- A thematic title rather than a raw filename
- Clear source attribution
- One grounded quote or evidence fragment when appropriate
- 2–5 sentences of unique analysis with validated links
- A connection to the day’s context or another source

## Anti-Patterns
- Template leakage
- Repetitive phrasing
- Hallucinated sources
- Invalid wikilinks
- Low cross-source synthesis density
